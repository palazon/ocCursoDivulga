DAFO — Presentación OpenCode

Taller 2h · Facultad de Biología, Universidad de Murcia · 2026-07-02

Debilidades

  • Parte 2 más esquemática que Parte 1: slides como "Análisis de resultados" o "Incluir documentación" con menos desarrollo
  • Parte 3 casi duplicada (MCP añadida) pero Skills y Git siguen siendo una slide cada uno
  • Tabla inicial "¿Qué es OpenCode?" usa jerga técnica (LSP, BYOK, multi-sesión) para público no programador
  • Sin indicación visual de profundidad con navigationMode: linear
  • ~35 slides para 2h — riesgo de sobrecarga, sobre todo si se abren debates

Amenazas

  • Evolución rápida de OpenCode: UI, beta Desktop App, modelos pueden quedar obsoletos
  • Enlaces externos (opencode.ai/docs/, palazon.github.io/…) pueden cambiar o caer
  • Tiempo limitado (2h) vs ~35 slides — riesgo de no cubrir Partes 2 y 3
  • Dependencia de conexión para ejemplos con Wikipedia y enlaces

Fortalezas

  • Diseño visual cuidado y coherente: fondo oscuro, verde institucional, callouts dorados, hover en tablas
  • Estructura pedagógica sólida: cada ejemplo sigue planteamiento + reflexión, patrón extendido a Parte 2
  • Tono cercano y eficaz: "tú", coloquial pero preciso, sin jerga innecesaria en ejemplos prácticos
  • Gradación de complejidad: Parte 1 (básico) → Parte 2 (proyecto) → Parte 3 (agentes/skills/git)
  • Ejemplo de agente contextualizado (revisor-coherencia) con YAML colapsable
  • Flujo de trabajo reescrito en tres fases (Plan → Build → Tú), más pedagógico que la versión anterior
  • Nuevas slides contextuales: barreras al cambio, MCP, referencias — amplían el alcance sin perder foco
  • CSS refinado: footnotes visibles, slide-number legible, código rosa, kbd estilizado
  • Corrección lingüística general: concordancias, mayúsculas, puntuación, ortografía
  • Tablas con tbl-colwidths en todas las tablas clave
  • Callouts sin título ni icono, con estilo propio que unifica la presentación

Oportunidades

  • OpenCode es gratuito y open source — sin barrera económica
  • Tendencia IA: el público ya tiene curiosidad, necesita ver aplicaciones concretas
  • Potencial de adaptación: ejemplos Wikipedia/Biología conectan con la facultad; extender a casos reales
  • Documentación GitHub Pages ya desplegada — enlaces directos a ampliación de Agentes y Skills
  • Slide MCP abre la puerta a conectar fuentes institucionales (glosarios, normativas, catálogos)
  • Público repetidor de tareas (guías, informes, actas cada curso) — alta probabilidad de adopción
