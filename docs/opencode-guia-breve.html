Qué es OpenCode

OpenCode es un asistente de IA que escribe, depura, modifica y revisa contenido por ti: código, documentos, textos, esquemas. Piensa en él como un colaborador que trabaja directamente en tu terminal.

Característica Detalle
Gratuito y código abierto Sin suscripciones ni costes ocultos
Funciona con varios proveedores Claude, GPT, Gemini, modelos locales
Trabaja desde la terminal Sin necesidad de interfaz gráfica
Se integra en tu editor Cualquier editor que soporte terminal
Conecta con GitHub Automatiza issues y pull requests

1. Instalación

Método rápido:

curl -fsSL https://opencode.ai/install | bash

Por gestor de paquetes:

# macOS
brew install opencode

# npm
npm i -g opencode-ai@latest

# Windows (Scoop)
scoop install opencode

App de escritorio: descarga desde opencode.ai/download.

2. Lanzar OpenCode

Una vez instalado, abre una terminal en la carpeta de tu proyecto y ejecuta:

cd tu-proyecto
opencode

OpenCode ya viene con un modelo gratuito predefinido (Zen). No necesitas configurar nada para empezar. Si quieres usar otro modelo o proveedor, puedes cambiarlo con el comando /models (consulte la sección 4).

3. Ajustes

Modos de trabajo

  • Modo Plan (Tab): solo lectura. Analiza y sugiere sin tocar archivos.
  • Modo Build (por defecto): ejecuta cambios, escribe archivos, corre comandos.

Regla práctica: contenido → Plan primero luego Build.

Archivo AGENTS.md

AGENTS.md es un archivo que le da contexto a OpenCode sobre tu proyecto: qué herramientas usas, qué estilo de redacción sigues y cómo están organizados tus archivos. Sin él, OpenCode funciona pero no conoce bien tu proyecto.

Puedes crearlo de dos formas:

  1. Manualmente: crea un archivo AGENTS.md en la raíz del proyecto.
  2. Con el comando /init: OpenCode te guía paso a paso para generarlo.

Ejemplo de contenido:

# Proyecto: Blog corporativo

## Stack
- Quarto para publicación
- Markdown como formato base
- HTML de salida

## Convenciones
- Tono cercano y profesional
- Párrafos cortos (máx. 3 líneas)
- Subtítulos descriptivos
- Imágenes con alt text

## Estructura
/_quarto.yml — configuración del proyecto
/styles.css — estilos personalizados
/posts — artículos del blog
/images — imágenes del sitio

Comandos esenciales

Comando Función
Tab Cambiar entre Plan y Build
/undo Deshacer último cambio
/redo Rehacer cambio
@ Buscar archivos en el proyecto
/compact Resumir la conversación actual
! Ejecutar un comando de terminal
/exit Salir de OpenCode (también /quit o /q)
/export Exportar la conversación a Markdown
/sessions Listar y volver a sesiones anteriores
/init Crear o actualizar AGENTS.md
/help Mostrar la tabla de ayuda

Modo no interactivo

Útil para scripting o automatización: ejecuta una orden sin abrir la interfaz, devuelve el resultado y cierra.

opencode run "Resume este documento en 5 puntos clave"

4. Modelos gratuitos

OpenCode incluye modelos gratuitos para empezar a usar sin necesidad de clave API. La oferta de modelos gratuitos puede cambiar, así que puedes pedirle a OpenCode una tabla actualizada de los disponibles.

Modelo Nota
Grok Code Fast 1 Gratuito por tiempo limitado
GLM 4.7 Gratuito por tiempo limitado
Big Pickle Gratuito por tiempo limitado

Puedes cambiar de modelo en cualquier momento con el comando /models.

5. Configuración (avanzado)

OpenCode funciona sin configuración. Solo necesitas elegir un modelo al iniciar. Si quieres usar tu propia clave API (por ejemplo, para acceder a modelos de pago o más potentes), sigue estos pasos.

Método guiado

Ejecuta el asistente de autenticación:

opencode auth login

OpenCode te guiará para seleccionar proveedor y entrar tu clave API.

Método manual

Configura la variable de entorno correspondiente:

# Anthropic
export ANTHROPIC_API_KEY="tu-clave"

# OpenAI
export OPENAI_API_KEY="tu-clave"

# Google
export GEMINI_API_KEY="tu-clave"

Archivo de configuración

Crea opencode.json en la raíz del proyecto (o en ~/.config/opencode/ para configuración global):

{
  "provider": "anthropic",
  "model": "claude-sonnet-4-20250514"
}

7. Consejos

  1. Sé específico: en vez de “mejora este texto”, escribe “acorta el párrafo de introducción a un máximo de 2 líneas y añade un gancho”.
  2. Da contexto: indica el público objetivo, el tono deseado y el formato de salida.
  3. Usa modo Plan, para evaluar o proponer antes de generar documentos o proyectos.
  4. Soporta imágenes: arrastra capturas o maquetas a la terminal para que OpenCode las analice, depende del modelo.
  5. Configuración por proyecto: usa opencode.json local para distintos repos de contenido, o deja que el propio OpenCode se encargue.
